# 自行车 Fitting 指导软件 — 设计文档

**日期**：2026-03-13
**状态**：已确认

---

## 概述

一款运行在浏览器中的公路车 fitting 指导工具，使用笔记本摄像头实时捕捉骑行姿态，通过姿势检测分析三项核心 fitting 指标，以视觉叠加和语音提示的方式实时引导用户调整坐姿。面向个人骑手居家使用，无需安装，无需后端服务。

---

## 目标用户

- 个人公路车骑手
- 无专业 fitting 知识，希望自助完成基本 fitting 校准
- 使用场景：室内骑行台（trainer）训练时

---

## 技术栈

| 层 | 选择 |
|---|---|
| 构建工具 | Vite |
| UI 框架 | React |
| 姿势检测 | MediaPipe Tasks Vision（`@mediapipe/tasks-vision` v0.10.x）· PoseLandmarker · lite 模型 |
| 画面叠加 | Canvas API（叠加在 `<video>` 元素上方） |
| 语音反馈 | Web Speech API（SpeechSynthesis，Chrome/Edge；Firefox/Safari 降级为静默模式） |
| 数据持久化 | localStorage |
| 部署 | 静态文件，无后端 |

**浏览器支持**：Chrome/Edge（完整功能）。Firefox 和 Safari 的 WebAssembly SIMD 支持存在不确定性，定为"不支持"，访问时显示浏览器不兼容提示。

**MediaPipe 说明**：使用新版 Tasks Vision API（`PoseLandmarker`），lite 模型在 MacBook 上可稳定达到 30fps。WASM 模型文件（约 6MB）随应用一起打包部署（不依赖外部 CDN），支持离线使用。关键点置信度阈值设为 0.5，低于此值的关键点不参与角度计算。

## 页面与用户流程

### 1. 设置页

- 请求摄像头权限
- 提示最佳拍摄位置：**侧面**（90° 侧视角），距离约 2-3 米，摄像头与骑手腰部等高
- 显示示意图说明正确 vs 错误拍摄角度（如头对着摄像头角度无法测量）
- 选择可用摄像头设备（如有多个）
- MediaPipe WASM 模型在此页面后台加载（约 6MB），显示加载进度条，加载完成后才能进入下一步

### 2. 校准页

- 实时显示摄像头画面
- MediaPipe PoseLandmarker 持续检测人体关键点
- 检测到完整骨架后显示绿色确认指示
- **稳定性判定**：连续 60 帧（约 2 秒）内，髋、膝、踝关键点的**归一化坐标**（MediaPipe 输出 0-1 范围）标准差均 < 0.01，且置信度均 > 0.5，则认为姿态稳定，自动进入主界面
- **超时处理**：若 30 秒内未满足稳定条件，显示"跳过"按钮，允许用户手动进入主界面
- 校准不验证摄像头角度（侧视 vs 正视），依赖设置页的文字引导；不正确角度会导致测量值失真，界面对此无警告

### 3. 主骑行界面

**布局**：左右分栏

- **左侧（2/3 宽）**：摄像头实时画面 + Canvas 骨架叠加
  - 关节连线颜色：绿色（正常）/ 黄色（边缘）/ 红色（偏离）
  - 三处角度数值以浮动标签显示在对应关节旁
- **右侧（1/3 宽）**：三项指标卡片，竖向排列
  - 指标名称 + 当前角度数值（大字）
  - 进度条显示当前值在目标范围内的位置
  - 目标范围文字说明

**退出**：按空格键或 ESC 停止录制，进入总结页

### 4. Session 总结页

- 本次骑行的三项指标平均值 vs 目标范围
- 综合评分显示为 0-100 整数（如"83 分"）
- 3-5 条文字调整建议（基于偏离方向，见下方建议映射表）
- 保存本次 session 数据到 localStorage（时间戳 + 平均角度）
- 可查看历史 session 列表（简单列表，显示日期和综合评分）

## 指标定义

### 膝盖角度（Knee Extension Angle）

- **计算**：髋关节 → 膝关节 → 踝关节 三点夹角
- **测量时机**：踩踏最低点（BDC）附近采样。BDC 检测方式：追踪踝关节的归一化 Y 坐标（MediaPipe 输出，Y 轴向下为正），取局部最大值时刻即为 BDC，在该时刻前后各 3 帧（共约 7 帧）取平均角度作为一次有效测量。此为屏幕空间启发式检测，摄像头需大致水平；低踏频或停止踩踏时可能漏测，届时角度显示维持上次有效值。
- **初始状态**：进入主骑行界面后，尚未检测到第一次 BDC 前，膝盖角度卡片显示"--"，进度条灰色，不触发语音反馈。
- **侧面遮挡处理**：摄像头从侧面拍摄时，优先使用靠近摄像头一侧的腿（通常是 MediaPipe 中 visibility 更高的一侧）；若两侧 visibility 均低于 0.5 则跳过本次测量。
- **目标范围**：140° – 150°
- **偏低（< 140°）**：坐垫偏低，建议升高坐垫
- **偏高（> 150°）**：坐垫偏高或后移，建议降低坐垫

### 躯干角度（Torso Angle）

- **计算**：肩关节中点 → 髋关节中点 连线，与屏幕垂直方向（Y 轴）的夹角
  - 肩中点 = (landmark[11] + landmark[12]) / 2（LEFT_SHOULDER, RIGHT_SHOULDER）
  - 髋中点 = (landmark[23] + landmark[24]) / 2（LEFT_HIP, RIGHT_HIP）
- **角度约定**：0° = 完全直立（肩-髋连线平行于 Y 轴），90° = 水平前倾（肩-髋连线平行于 X 轴）。角度越小越直立，越大越前倾。
- **垂直基准**：使用屏幕空间垂直方向（假设摄像头水平放置）。设置页提示用户将摄像头调平，并在校准页显示一条参考水平线辅助确认。
- **测量时机**：持续采样，对最近 30 帧取滑动平均值
- **目标范围**：35° – 45°（相对垂直）
- **偏小（< 35°）**：上体过于前倾，可能需要调高把立或缩短车把距
- **偏大（> 45°）**：上体过于直立，可能需要降低把立或增加前伸量

### 手肘角度（Elbow Angle）

- **计算**：肩关节 → 肘关节 → 腕关节 三点夹角
- **侧面遮挡处理**：摄像头从侧面拍摄时，近侧手臂遮挡远侧，只测量置信度 > 0.5 的一侧（通常为靠近摄像头的手臂）。若两侧均低于阈值，则本帧跳过，不更新显示值。
- **测量时机**：持续采样，对最近 30 帧（置信度通过的帧）取滑动平均值
- **目标范围**：150° – 165°
- **偏小（< 150°）**：手肘弯曲过多，可能车把距偏短
- **偏大（> 165°）**：手肘接近伸直，可能车把距偏长或把立偏高

## 调整建议映射表

总结页根据以下规则生成建议（仅显示偏离的项目）：

| 指标 | 方向 | 建议文本 |
|---|---|---|
| 膝盖角度 | < 140° | "膝盖伸展角度偏小，建议升高坐垫约 5-10mm" |
| 膝盖角度 | > 150° | "膝盖伸展角度偏大，建议降低坐垫约 5-10mm" |
| 躯干角度 | < 35° | "上体过于前倾，建议调高把立或缩短把立长度" |
| 躯干角度 | > 45° | "上体过于直立，建议降低把立或更换更长把立" |
| 手肘角度 | < 150° | "手肘弯曲过多，建议更换更长把立或调整把位" |
| 手肘角度 | > 165° | "手肘接近伸直，建议更换更短把立，保持轻微弯曲" |

若三项均在目标范围内，显示："当前 fitting 状态良好，继续保持！"

### 视觉反馈

- Canvas 实时重绘，帧率跟随 MediaPipe 检测帧率（目标 30fps）
- 骨架关键线段按指标状态着色：
  - 绿色：在目标范围内
  - 黄色：偏离 5° 以内（边缘警告）
  - 红色：偏离超过 5°
- 角度数值标签实时更新，显示在对应关节附近

### 语音反馈

- 使用 Web Speech API（中文语音）
- **浏览器兼容**：Chrome/Edge 完整支持；Firefox 的 SpeechSynthesis 支持不稳定，降级为静默模式（仅视觉反馈，界面显示"语音不可用"提示）
- 触发条件：某项指标偏离目标范围超过 **3 秒**
- 去重：同一条提示 **30 秒** 内不重复播报（每项指标独立计时）
- 优先级：膝盖角度 > 躯干角度 > 手肘角度（同时偏离时先播报最重要的）
- 示例播报内容：
  - "膝盖角度偏小，建议升高坐垫"
  - "上体过于直立，注意前倾姿势"
  - "手肘弯曲过多，尝试放松手臂"

## 数据模型（localStorage）

```typescript
interface Session {
  id: string;           // UUID
  timestamp: number;    // Unix timestamp
  duration: number;     // 骑行秒数
  avgKneeAngle: number;
  avgTorsoAngle: number;
  avgElbowAngle: number;
  score: number;        // 综合评分 0-100，见下方公式
}

// 评分公式：
// 每项指标计算单项得分（0-100），三项取加权平均
// 单项得分 = max(0, 100 - (abs(actualAngle - midpoint) / halfRange) * 100)
//   midpoint = (lowerBound + upperBound) / 2
//   halfRange = (upperBound - lowerBound) / 2
// 偏离 = 0（在目标中点）→ 100 分；偏离 ≥ halfRange（超出范围边界）→ 0 分；中间线性插值
// 权重：膝盖 50%，躯干 30%，手肘 20%

// localStorage key: "bike-fitting-sessions"
// 存储格式: Session[]，最多保存 20 条（超出时删除最旧的）
```

## 组件结构

`CameraView` 和 `PoseLandmarker` 实例在 `App` 层通过 React Context 共享，页面切换时不重建 MediaPipe pipeline。

```
App（持有 CameraContext 和 PoseContext）
├── SetupPage
│   ├── CameraPermissionRequest
│   └── CameraPositionGuide
├── CalibrationPage          // 消费 CameraContext + PoseContext
│   └── SkeletonOverlay
├── RidingPage               // 消费 CameraContext + PoseContext
│   ├── SkeletonOverlay      // Canvas 叠加骨架
│   ├── AngleCalculator      // 角度计算逻辑（custom hook）
│   ├── MetricsPanel         // 右侧三指标卡片
│   └── VoiceFeedback        // 语音播报逻辑
└── SummaryPage
    ├── MetricsSummary
    ├── Recommendations
    └── SessionHistory
```

## 错误处理

| 场景 | 处理方式 |
|---|---|
| 摄像头权限被拒 | 设置页显示错误说明和重试按钮，无法进入后续步骤 |
| MediaPipe WASM 加载失败 | 显示错误页，提示检查网络连接或使用 Chrome |
| 骨架检测置信度不足 | 校准页保持等待状态，提示"请确保身体完整入镜，光线充足" |
| Web Speech API 不可用 | 静默降级，界面右上角显示"🔇 语音不可用"标识 |
| localStorage 写入失败（配额超限） | 静默忽略，Session 数据不保存，不影响核心骑行体验 |

## 成功标准

1. 在 Chrome/Edge 中无需安装即可运行（Firefox/Safari 显示不兼容提示）
2. MediaPipe 检测帧率 ≥ 25fps（MacBook 2019+ 环境）
3. 角度计算误差目标 < 5°，通过与静态图片手工测量对比验证
4. 一次 10 分钟骑行 session 中，每项指标语音播报最多 ~20 次（30 秒冷却），总次数上限约 60 次；实际体验目标为偏离时提醒不超过每 30 秒 1 次
5. 初次使用用户（无引导）能在 3 分钟内开始第一次骑行分析
